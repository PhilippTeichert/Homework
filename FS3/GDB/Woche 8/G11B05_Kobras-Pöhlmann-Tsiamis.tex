\subsection{Teilaufgabe c}
Das Schema ist in folgenden Fällen unsicher bezüglich referentiellen Aktionen:\\
Ein Eintrag der Tabelle "BenutzerIn" wird gelöscht.\\
Aktualisiert man zuerst die Tabelle "RubrikZuordnung\string" und danach erst die Tabelle "Rubriken", kann dies dazu führen, dass sich Rubriken jetzt löschen lassen, die auf anderem Wege abgelehnt worden wären.

%---------------------------------------------------------------

\subsection{Teilaufgabe d}
Dieser Missstand lässt sich z.B. dadurch beheben, dass man das \textbf{ON DELETE RESTRICT} zwischen "Rubriken\string" und "RubrikZuordnung\string" in ein \textbf{ON DELETE CASCADE} umändert.\\
Andernfalls könnte man auch die beiden \textbf{ON DELETE CASCADE} zwischen "BenutzerIn\string" und "Rubriken\string" und zwischen "BenutzerIn\string" und "RubrikZuordnung\string" jeweils in ein \textbf{ON DELETE RESTRICT} umändern.

\section{Änderbarkeit von Sichten}
\subsection{Teilaufgabe a}
\subsubsection{i}

\begin{verbatim}
CREATE VIEW EnterpriseCrew
  AS SELECT BNr, Name, Rang
    FROM Besatzungsmitglieder
    WHERE IN
      (SELECT * FROM Besatzungsmitglieder, Raumschiffe
        WHERE Besatzungsmitglieder.Schiff = Raumschiffe.RNr
        AND Raumschiffe.Name = 'Enterprise')
        WITH CASCADED CHECK OPTION;
\end{verbatim}
Diese Relation ist änderbar.
Deshalb wurde eine \textbf{CHECK OPTION} eingefügt.

\subsubsection{ii}
\begin{verbatim}
CREATE VIEW Captains
  AS SELECT Name
    FROM Besatzungsmitglieder
    WHERE Besatzungsmitglieder.Rang = 'Captain';
\end{verbatim}
Diese Relation ist nicht änderbar, da der Primärschlüssel fehlt.

\subsubsection{iii}
\begin{verbatim}
CREATE VIEW WarpFed
  AS SELECT RNr, Fraktion, Baujahr
    FROM Raumschiffe
    WHERE Raumschiffe.Geschwindigkeit >= 1
    AND Raumschiffe.Fraktion = 'Föderation'
    WITH CASCADED CHECK OPTION;
\end{verbatim}
Diese Relation ist änderbar.
Deshalb wurde eine \textbf{CHECK OPTION} eingefügt.

\subsection{Teilaufgabe b}
Es existieren folgende Abhängigkeiten:
\begin{itemize}
    \item T\textsubscript{1} liest A und T\textsubscript{2} schreibt A.
    \item T\textsubscript{2} liest A und T\textsubscript{1} schreibt A.
    \item T\textsubscript{1} liest B und T\textsubscript{2} schreibt B.
\end{itemize}
Daraus folgt, dass verschiedene Werte für A und B raus kommen, je nachdem welche dieser 6 Aktionen an welcher Stelle kommt.\\
So greifen z.B:\\
w\textsubscript{1}(A), r\textsubscript{1}(A), w\textsubscript{2}(A), r\textsubscript{2}(A)\\
alle auf das erste Feld zu.\\
Und\\
w\textsubscript{1}(B), r\textsubscript{2}(B)\\
beide auf das zweite Feld.\\
\\
Dies bedeutet für die folgenden Schedules:
\begin{itemize}
    \item S\textsubscript{1}
    \begin{itemize}
        \item Abhängigkeit(w\textsubscript{1}(A) vor r\textsubscript{2}(A))
    \end{itemize}
    
    \item S\textsubscript{2}
    \begin{itemize}
        \item Abhängigkeit(r\textsubscript{1}(A) vor w\textsubscript{2}(A))
        \item Abhängigkeit(r\textsubscript{1}(B) vor w\textsubscript{2}(B))
        \item Abhängigkeit(w\textsubscript{2}(A) vor w\textsubscript{1}(A))
    \end{itemize}
    
    \item S\textsubscript{3}
    \begin{itemize}
        \item Abhängigkeit(w\textsubscript{2}(B) vor r\textsubscript{1}(B))
        \item Abhängigkeit(w\textsubscript{2}(A) vor r\textsubscript{1}(A))
    \end{itemize}
    
    \item S\textsubscript{4}
    \begin{itemize}
        \item Abhängigkeit(w\textsubscript{2}(B) vor r\textsubscript{1}(B))
        \item Abhängigkeit(r\textsubscript{1}(A) vor w\textsubscript{2}(A))
        \item Abhängigkeit(w\textsubscript{2}(A) vor w\textsubscript{1}(A))
    \end{itemize}
    
    \item S\textsubscript{5}
    \begin{itemize}
        \item Abhängigkeit(r\textsubscript{2}(A) vor w\textsubscript{1}(A))
        \item Abhängigkeit(w\textsubscript{1}(A) vor w\textsubscript{2}(A))
    \end{itemize}
    
    \item S\textsubscript{6}
    \begin{itemize}
        \item Abhängigkeit(w\textsubscript{2}(B) vor r\textsubscript{1}(B))
        \item Abhängigkeit(w\textsubscript{2}(A) vor r\textsubscript{1}(A))
    \end{itemize}
\end{itemize}

\subsection{Teilaufgabe c}
Die Zeitpläne sind:
\begin{itemize}
    \item S\textsubscript{1}
    \begin{itemize}
        \item seriell, da erst alle Operationen von T\textsubscript{1} und danach alle Operationen von T\textsubscript{2} ausgeführt werden\\
        (da Abhängigkeit(w\textsubscript{1}(A) vor r\textsubscript{2}(A)) gilt).
    \end{itemize}
    
    \item S\textsubscript{2}
    \begin{itemize}
        \item nicht serialisierbar, da T\textsubscript{1} den Grundwert liest und danach das Ergebnis von T\textsubscript{2} überschreibt\\(da Abhängigkeit(w\textsubscript{2}(A) vor w\textsubscript{1}(A)) und Abhängigkeit(r\textsubscript{1}(A) vor w\textsubscript{2}(A)) gilt).
        \item Datenanomalie: T\textsubscript{2} hat in den Daten
        nicht statt gefunden.
    \end{itemize}
    
    \item S\textsubscript{3}
    \begin{itemize}
        \item serialisierbar, da T\textsubscript{1} erst liest, nachdem T\textsubscript{2} sie nicht mehr weiter bearbeitet\\
        (da Abhängigkeit(w\textsubscript{2}(B) vor r\textsubscript{1}(B)) und Abhängigkeit(w\textsubscript{2}(A) vor r\textsubscript{1}(A)) gilt).
    \end{itemize}
    
    \item S\textsubscript{4}
    \begin{itemize}
        \item nicht serialisierbar, da die Veränderung des Wertes A von T\textsubscript{2} nicht in der Datenbank auftaucht\\
        (da Abhängigkeit(r\textsubscript{1}(A) vor w\textsubscript{2}(A)) und Abhängigkeit(w\textsubscript{2}(A) vor w\textsubscript{1}(A)) gilt).
        \item Datenanomalie: T\textsubscript{2} verändert in den Daten nur B nicht jedoch A.
    \end{itemize}
    
    \item S\textsubscript{5}
    \begin{itemize}
        \item nicht serialisierbar, da die Veränderung des Wertes A von T\textsubscript{1} nicht in der Datenbank auftaucht\\
        (da Abhängigkeit(r\textsubscript{2}(A) vor w\textsubscript{1}(A)) und Abhängigkeit(w\textsubscript{1}(A) vor w\textsubscript{2}(A)) gilt).
        \item Datenanomalie: T\textsubscript{1} hat in den Daten nicht statt gefunden.
    \end{itemize}
    
    \item S\textsubscript{6}
    \begin{itemize}
        \item seriell, da erst T\textsubscript{2} und danach T\textsubscript{1} ausgeführt wird\\
        (da Abhängigkeit(w\textsubscript{2}(B) vor r\textsubscript{1}(B)) und Abhängigkeit(w\textsubscript{2}(A) vor r\textsubscript{1}(A)) gilt).
    \end{itemize}
\end{itemize}
